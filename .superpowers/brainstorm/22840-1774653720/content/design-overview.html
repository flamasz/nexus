Design Overview: Revision 1

Adapt existing packaging components for reuse across Order Items and the entire app

1. Component Changes

📝 ItemNameCombobox
Add: variant="compact" prop
Compact mode: Smaller trigger, no label, pill badge stays, dropdown unchanged
Used in: PackagingForm (default) + OrderItemRow (compact)
🏷️ CategorySelector
Add: variant="compact" prop
Compact mode: Smaller trigger with color dot + name, no dimensions info box below, dropdown unchanged with colors/dimensions/edit
Used in: PackagingForm (default) + OrderItemRow (compact)
🆕 VersionCombobox
New component modeled after ItemNameCombobox
Props: itemNameId, categoryId, versions list, onSelect, onCreate
Behavior: Searchable dropdown, "+ Create new version" at bottom, disabled when item_name or category is null
Supports: default + compact variants
🎨 ItemStatusDropdown
Already exists and works as a color-coded pill with dropdown
Add: variant="compact" prop for smaller sizing
Used for: Approval Status in OrderItemRow
🚦 ItemOrderStatusDropdown
New component modeled after ItemStatusDropdown
Statuses: new (red), final (green), cancel (gray) — same colors as current
Behavior: Color-coded pill trigger, colored dropdown options on click
Supports: default + compact variants
🔢 QTY Field
Simple CSS fix: Hide spinner buttons via CSS
input[type=number]::-webkit-inner-spin-button { display: none; }
Applied globally or as a utility class

2. OrderItemRow Rewrite

Replace all 4 <select> elements in OrderItemRow with:
  • <ItemNameCombobox variant="compact" /> → replaces Item Name select
  • <CategorySelector variant="compact" /> → replaces Category select
  • <VersionCombobox variant="compact" /> → replaces Version select
  • <ItemStatusDropdown variant="compact" /> → replaces Approval Status select
  • <ItemOrderStatusDropdown variant="compact" /> → replaces Item Order Status select
  • QTY <input> stays, just hide spinners
